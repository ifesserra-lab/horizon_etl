# Documenting ETL Flows

Follow these steps when implementing a new data source or export:

1. **Update Requirements**: Add functional requirements to `docs/2 - implementacao/SI1-Requisitos.md`.
2. **Draft Design**: Use the `etl_design_doc.md` template to create a new document in `docs/architecture/` (or update appropriate sections in SDD).
3. **Map Transformations**: Ensure the Mapping Matrix is detailed in the design doc.
4. **Define Verification**: Document the exact commands to verify data integrity.
5. **Update Implementation Plan**: Include documentation as a checklist item in `implementation_plan.md`.
6. **Final Walkthrough**: Reference the design implementation and verification results in `walkthrough.md`.
