# Implementation Plan: Containerized Application Deployment

**Branch**: `002-docker-compose-app` | **Date**: 2026-05-16 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/002-docker-compose-app/spec.md`

## Summary

Add a `Dockerfile` for the Horizon ETL app and extend `docker-compose.yml` with an `app` service, enabling the full system (Prefect DB + Prefect server + ETL app) to start with `make docker-up` and run individual flows via `make docker-pipeline`, `make docker-ingest-sigpesq`, etc. Data is persisted through bind mounts so the developer can inspect SQLite, exports, and logs directly from the host.

## Technical Context

**Language/Version**: Python 3.12 (matches existing venv)

**Primary Dependencies**: Docker Engine + Compose v2, `python:3.12-slim` base image, `chromium-browser` + `chromium-chromedriver` (apt), Playwright `--with-deps chromium`

**Storage**: SQLite (`db/horizon.db`) via bind mount; PostgreSQL (Prefect metadata) via named volume

**Testing**: Existing `pytest` suite; manual integration via quickstart scenarios

**Target Platform**: Linux container (amd64), macOS host (developer workstation)

**Project Type**: ETL pipeline вҖ” run-on-demand containerized CLI tool

**Performance Goals**: All services healthy within 2 minutes after images pulled (SC-001); first-time setup within 10 minutes including image pull (SC-002)

**Constraints**: Chromium must run headless inside container; `.env` credentials must never be baked into the image; all LGPD anonymization must remain active (LGPD hook in `app.py` runs unchanged)

**Scale/Scope**: Single developer local machine

### Dockerfile (multi-stage)

```dockerfile
# Stage 1: Build Python deps
FROM python:3.12-slim AS builder
WORKDIR /build
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential git && rm -rf /var/lib/apt/lists/*
COPY requirements.txt .
RUN pip install --user --no-cache-dir -r requirements.txt

# Stage 2: Runtime
FROM python:3.12-slim
WORKDIR /app

# Chromium + Playwright system deps
RUN apt-get update && apt-get install -y --no-install-recommends \
    chromium chromium-driver \
    libglib2.0-0 libnss3 libnspr4 libdbus-1-3 \
    libx11-6 libxext6 libxrender1 libxkbcommon0 \
    libatk1.0-0 libatk-bridge2.0-0 \
    libpango-1.0-0 libpangocairo-1.0-0 libcairo2 libasound2 \
    curl && \
    rm -rf /var/lib/apt/lists/*

COPY --from=builder /root/.local /root/.local
ENV PATH=/root/.local/bin:$PATH \
    PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    PYTHONPATH=/app \
    CHROME_BINARY=/usr/bin/chromium \
    CHROMEDRIVER_PATH=/usr/bin/chromedriver

COPY . /app

# Install Playwright's own Chromium + deps
RUN python -m playwright install --with-deps chromium

ENTRYPOINT ["python", "app.py"]
CMD ["--help"]
```

## Implementation Notes

### ChromeDriver path override in scriptLattes

The existing `chromedriver` binary in the repo root is a macOS arm64 binary. Inside the container, `/usr/bin/chromedriver` (from apt) must be used. The `CHROMEDRIVER_PATH` env var must be read by the Lattes download flow. Check `src/flows/lattes/download.py` for chromedriver path configuration and ensure it reads `os.environ.get("CHROMEDRIVER_PATH", "./chromedriver")`.

### Database directories

Bind mount target directories must exist on the host before the container starts. The `make docker-up` target should run `mkdir -p db data/exports data/lattes_json cache logs` before invoking `docker compose up`.

### Prefect ephemeral mode disabled in container

Inside the container, `PREFECT_API_URL=http://server:4200/api` points to the real Prefect server. Remove `PREFECT_SERVER_ALLOW_EPHEMERAL_MODE` from the container environment вҖ” it must connect to the server, not run ephemerally.
