# Implementation Plan: Weekly CI Workflow Validation and Export Compression

**Branch**: `feat/003-workflow-ci-zip-exports` | **Date**: 2026-05-16 | **Spec**: [spec.md](spec.md)

## Summary

Fix the `weekly-etl.yml` GitHub Actions workflow to resolve three issues: missing `actions: write` permission (blocks artifact upload), absent Chrome/chromedriver for the Lattes Selenium flow, and uncompressed exports exceeding the 10 MB threshold. All changes are confined to a single YAML file.

**Storage**: N/A — workflow produces ephemeral CI artifacts (30-day retention)

**Testing**: Manual `workflow_dispatch` trigger on the `weekly-etl` workflow

**Target Platform**: GitHub Actions ubuntu-latest

**Project Type**: CI/CD workflow configuration

**Performance Goals**: Artifact upload size reduced ≥30% when exports exceed 10 MB

**Constraints**: Single file change (`.github/workflows/weekly-etl.yml`); no src/ modifications

**Scale/Scope**: One weekly schedule + manual dispatch; ~4 runs/month

## Changes in Detail

### 1. Add `actions: write` permission

```yaml
permissions:
  contents: read
  actions: write        # required by upload-artifact@v4
```

### 2. Install Chromium apt pair for Lattes (after Playwright install step)

```yaml
- name: Install Chromium for Lattes (Selenium)
  run: |
    sudo apt-get update -qq
    sudo apt-get install -y chromium-browser chromium-driver
```

Add env vars at job level:

```yaml
env:
  CHROME_BINARY: /usr/bin/chromium-browser
  CHROMEDRIVER_PATH: /usr/bin/chromedriver
```

### 3. Compress exports >10 MB before upload

```yaml
- name: Compress large export files
  if: always()
  run: |
    find data/ -type f -size +10M \
      -exec sh -c 'for f; do zip -m -j "$f.zip" "$f"; done' sh {} +
```

### 4. Add `retention-days: 30` to artifact upload

```yaml
- name: Upload ETL artifacts
  if: always()
  uses: actions/upload-artifact@v4
  with:
    name: weekly-etl-artifacts
    retention-days: 30
    if-no-files-found: ignore
    path: |
      data
      db/horizon.db
      !.venv
      !.venv/**
      !.env
      !.env.*
```
